# Code of Conduct

Shelter should be a welcoming, constructive, and accessible place to collaborate. This applies to issues, pull requests, reviews, discussions, and every other project space.

## Expected behavior

- Treat others with respect and assume good intent.
- Critique ideas and code specifically, never the person behind them.
- Keep discussions understandable and focused on the topic.
- Respect different levels of experience, backgrounds, identities, and communication styles.
- Accept feedback, correct mistakes openly, and make space for others.

## Unacceptable behavior

Harassment, insults, threats, discriminatory language, sexualized attention, doxxing, deliberate disruption, and publishing private information without consent are not tolerated.

## Enforcement

Project maintainers may edit or remove content and temporarily or permanently exclude participants when necessary to maintain a safe and constructive project space. Actions should be proportionate and account for context.

Report incidents confidentially to the project maintainers. If the incident also concerns a vulnerability or sensitive technical data, use the private reporting process in [SECURITY.md](SECURITY.md). Reports will remain as confidential as a fair investigation allows.
