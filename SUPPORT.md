# Support

Shelter is currently an MVP and is provided without guaranteed response or recovery times.

## Where should I ask?

- **Bug:** use the bug template and describe a reproducible problem.
- **Feature idea:** use the feature template and explain the concrete use case.
- **Operational question:** check [README.md](README.md), especially troubleshooting and the current MVP limitations.
- **Security vulnerability:** do not open a public issue. Follow [SECURITY.md](SECURITY.md).

## Information that helps

For technical problems, include when possible:

- the Shelter version or commit,
- the operating system and Docker and Compose versions,
- the source type and detected build type,
- exact steps leading to the problem,
- the expected and actual result,
- the smallest relevant excerpt of redacted logs.

Before sharing anything, remove passwords, tokens, OAuth parameters, cookies, private repository URLs, IP addresses, and other confidential data. A screenshot of the error is useful; a complete `.env` export never is.

Support generally targets the current state of the default branch. For older installations, first create a backup and review the documented update path.
